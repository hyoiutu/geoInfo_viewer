# Branch Rules (ブランチ運用規約)

本ファイルは、AIエージェントがGitHub Issueをもとに実装を進める`issue-implement`スキルを実行する際に遵守すべき、ブランチ運用の詳細なルールを定義します。

---

## 🚫 絶対遵守ルール (Mandatory Rules)

0. **1 Issue = 1 Branch**
   - 1つのGitHub Issueに対して、必ず1つの専用ブランチを割り当てること。
   - 複数のIssueを同一ブランチで同時に扱ってはならない。
   - 対象Issue用のブランチが既に存在する場合は新規作成せず、そのブランチをチェックアウトして再利用すること。

1. **プッシュの制限**
   - 対話モード（ユーザーとチャットでやり取りしながら1件のIssueを進める通常時）では、ユーザーから明示的な指示が無い限り`git push`を実行してはならない。リモートリポジトリへの送信は原則ユーザー（人間）自身が行う。
   - `issue-implement`スキルの**自律モード**（[issue-implement/SKILL.md](../.agents/skills/issue-implement/SKILL.md)参照）でのみ、1つのIssueの実装・検証・コミットが全て完了した後の最終ステップとして`git push`（通常のpush、`-u origin <ブランチ名>`）を行ってよい。
   - [finish-review](../.agents/skills/finish-review/SKILL.md)スキルでも、レビュー対応完了後の後処理として`git push`（およびPRのマージ）を行うが、マージの実行前には必ずユーザーにY/N確認を行う。本スキルは常に対話モードで実行し、自律モードは持たない。
   - **`git push --force`/`-f`（force push）は、対話モード・自律モードいずれであっても絶対に禁止。** このルールに例外は無い。

2. **ブランチ作成前のユーザー確認（対話モード）**
   - 対話モードでブランチを作成する前に、`issue-implement`スキルのプロセスに従い、以下をユーザーに確認し合意を得ること。
     - 対象とするGitHub Issue番号
     - ブランチ名
     - 派生元ブランチ
   - **自律モードでは、ユーザーの都度の応答を待てないため本確認を省略し、`issue-implement`スキルの自律モードの手順に従いAIが自己判断で決定する。**

3. **ブランチ名の決定**
   - ユーザーがブランチ名を明示的に指定した場合は、その名前をそのまま使用する。
   - 指定がない場合、AIは以下の形式でブランチ名を自動生成する。
     ```text
     <type>/issue-<Issue番号>-<内容を表す英語スラッグ>
     ```
     - `<type>`: [commit_rules.md](./commit_rules.md)のコミットメッセージプレフィックス規約（`feat`/`fix`/`docs`/`style`/`refactor`/`test`/`chore`）に準じて、Issueの内容から最も適切なものを選択する。
     - `<内容を表す英語スラッグ>`: Issueタイトルを要約した、英語の小文字・ハイフン区切り文字列（kebab-case）。
     - 例:
       - `feat/issue-3-mapbox-tile-source`
       - `fix/issue-7-map-drag-boundary`
       - `docs/issue-11-update-system-spec`

4. **派生元ブランチ**
   - 対話モードでは、デフォルトでスキル起動時にAIエージェントが作業しているカレントブランチから分岐する。ユーザーが別のブランチ（例: `main`）を指定した場合は、そちらから分岐する。派生元ブランチに未コミットの変更が残っている場合は、ブランチを切る前にユーザーに状況を伝え、確認を得ること（コミットする場合はauto-commitスキル経由とする）。
   - 自律モードでは、`issue-implement`スキルの自律モードの手順（直前のIssueが正常にPRを作成できていればそのブランチから縦方向に連鎖する。直前のIssueが行き詰まって中断した場合は、そのブランチではなくさらに1つ前の正常なブランチから分岐し、行き詰まったブランチの上には積まない）に従う。

5. **ブランチ作成コマンド**
   - 新規作成時: `git checkout -b <ブランチ名> <派生元ブランチ>`
   - 既存ブランチの再利用時: `git checkout <ブランチ名>`

6. **実装後のコミット**
   - ブランチ上で実装した変更をコミットする際は、必ず[auto-commitスキル](./commit_rules.md)の手順（本ファイルではなく`commit_rules.md`に従う）を経由すること。本スキルおよびAIエージェントが直接`git commit`を実行してはならない。
