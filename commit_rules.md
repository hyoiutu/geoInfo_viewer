# Commit Rules (コミット規約)

本ファイルは、AIエージェントが自動コミットスキルを実行する際に遵守すべき詳細なルールを定義します。

---

## 🚫 絶対遵守ルール (Mandatory Rules)

0. **スキル経由以外のコミット禁止**
   - git commitはauto-commitスキルの手順（本ファイルの遵守を含む）を経由した場合のみ実行してよい。スキルを起動せずに直接git commitを実行してはならない。

1. **プッシュの絶対禁止**
   - 本スキルまたはAIエージェントの処理を通じて、リモートリポジトリへのプッシュ (`git push`) を絶対に実行してはならない。

2. **テスト・リント・型チェックの無視禁止**
   - コミット前に、本ファイル「開発プロジェクト固有の検証手順」に記載された全てのコマンド（テスト・リント・型チェック・対象範囲に応じたE2Eテスト等）を必ず実行すること。
   - いずれかが失敗した場合、その結果を無視してコミットを進めてはならない。エラー内容をユーザーに報告し、修正を行うか、コミット処理を中断すること。

3. **コミットメッセージのプレフィックスと要約**
   - コミットメッセージは変更内容を要約し、1行程度（簡潔で明確）にまとめること。
   - 1行にまとめられない場合は変更規模が大きいのでコミットを分割すること。
   - コミットメッセージの先頭には、変更の種類を表す以下のプレフィックスを必ず付与すること。
     - `feat:` : 新機能やスキルの追加
     - `fix:` : バグや不具合の修正
     - `docs:` : ドキュメント（README, 仕様書, コミット規約など）のみの修正・追加
     - `style:` : コードの意味に影響を与えない変更（フォーマット、インデント、セミコロンの調整など）
     - `refactor:` : 新機能追加やバグ修正を含まない、コードのリファクタリング
     - `test:` : テストコードの追加・修正
     - `chore:` : ビルド構成やパッケージ追加などの雑多な変更
   - プレフィックスの直後には半角コロンと半角スペース（`: `）を配置し、続けて変更内容を1行程度（簡潔で明確）に要約すること。

4. **コミット前のユーザー確認**
   - コミットを実行する前に、以下の情報を必ずチャットに明示し、ユーザーから Y/N の承認を得ること。
     - コミットされるファイル一覧
     - コミットメッセージ案
     - 対象ブランチ名

## 📝 開発プロジェクト固有の検証手順

- **テストコマンド**: `pnpm run test:unit`（`pnpm --filter frontend test:unit && pnpm --filter backend test:unit`のエイリアス。単体テスト作成・実行時は test_rules.md のルールに従うこと）
- **リントコマンド**: `pnpm run lint`（Biome。対象は `electron/**`, `frontend/src/**`, `backend/src/**`。backend単体には専用のlintスクリプトはなく、ルートのBiome設定でカバーする）
- **型チェックコマンド**: `pnpm run typecheck`（`frontend/tsconfig.json`・`backend/tsconfig.json`・`tsconfig.electron.json`の3つを対象とする）
- **E2Eテストコマンド**: `pnpm run test:e2e`（`pnpm run build`によるビルドを含んだ上でPlaywrightを実行する。`electron/**`, `frontend/src/**`配下の実装ファイル（テストコードのみの変更を除く）に修正がある場合は必ず実行すること。README.md・仕様書・スキル定義等のドキュメントのみの修正の場合は実行不要。backend/はElectronメインプロセスからまだ起動されないため、backend/src/**のみの修正では現時点では対象外）

- **DB(PostgreSQL/PostGIS)を伴う変更のコミット前確認**: `backend/src/`にTypeORM Entity・マイグレーションの変更がある場合、単体テスト（Repositoryモック、test_rules.md参照）に加えて`pnpm --filter backend run migration:run`が実行可能なことを確認する（ローカルDB環境が無い場合はコミット者自身の環境で実行し、結果を報告する）。
