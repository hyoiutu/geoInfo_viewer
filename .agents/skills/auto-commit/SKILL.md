# AI自動コミットスキル (auto-commit)

このスキルは、AIエージェント自身がローカルファイルに変更を加えた際、自動的に安全なコミット処理を行うための指示書です。

## 🎯 起動条件

AIエージェントがタスクの実行中または完了時に、ローカルファイルに変更（`git diff` や `git status` で確認できる差分）が存在することを検知した場合、自動的にこのスキルを起動してワークフローを実行してください。

---

## 📋 実行プロセス（手順）

### 1. 差分の確認
- `git status` および `git diff` を使用して、どのようなファイルに変更が発生したかを確認します。

### 2. コミット規約の読み込み
- プロジェクトのルートディレクトリにある [commit_rules.md](../../../commit_rules.md) を読み込みます。
- 記載されているルール（特にプッシュの禁止、テスト・リントの無視禁止など）を再確認します。

### 3. テスト・リント・型チェックの検証
- コミットする前に、commit_rules.md の「開発プロジェクト固有の検証手順」に記載された**全てのコマンド**（テスト・リント・型チェック等）を実行し、すべて成功することを確認します。
- いずれかが失敗した場合は、コミットを中断し、ユーザーにエラーを報告して修正を行うか指示を仰いでください。決してエラーを無視してコミットを進めてはいけません。

### 4. コミットメッセージの生成
- 差分内容から、変更内容を要約した1行程度（簡潔で明確）のコミットメッセージを自動生成します。

### 5. ユーザーへの最終確認 (Y/N)（対話モードのみ）
- 通常（対話モード、ユーザーとチャットでやり取りしている最中）は、コミットを行う前に以下の情報を明確にチャットでユーザーに提示し、コミットして良いか「Y/N」で確認を求めてください。
  - **コミット対象ファイルの一覧**
  - **生成したコミットメッセージ案**
  - **現在の対象ブランチ名**
- **ユーザーが「Y」または肯定的な同意をするまで、決してコミットコマンド（`git commit`）を実行してはなりません。**
- **例外（自律モード）**: `issue-implement`または`pr-review-respond`スキルの自律モード（ユーザーが離席中の連続実行）から呼び出された場合に限り、本確認ステップは省略してよい。ただしこの場合も手順3の検証（テスト・lint・型チェック）が全て成功していることが前提であり、検証を省略してはならない。

### 6. コミットの実行
- 対話モードではユーザーから承認が得られた場合、自律モードでは検証が成功した時点で、以下のコマンドでコミットを実行します。
  - `git add <対象ファイル>`
  - `git commit -m "<メッセージ内容>"`
- コミット完了後、コミットハッシュや結果を（対話モードならユーザーに、自律モードなら後続の記録・PR説明に）反映します。

### 7. 🚫 【警告】本スキル単体でのプッシュ禁止
- **本スキル自体は、いかなるモードであっても`git push`を実行しません。** プッシュは`issue-implement`・`pr-review-respond`スキルの自律モードにおける、実装・レビュー対応が全て完了した後の専用ステップとしてのみ許可されています（詳細は[branch_rules.md](../../../branch_rules.md)参照）。本スキルの一部としてプッシュを行ってはいけません。
