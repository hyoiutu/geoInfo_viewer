# PRレビュー対応スキル (pr-review-respond)

このスキルは、AIエージェントがGitHub PRに寄せられたレビューコメントに対して、人間の都度の指示を介さずに一件ずつ誠実に対応を進めるための指示書です。

## 🎯 起動条件

ユーザーから「PRのレビューを確認して」「レビューコメントに対応して」等、GitHub PRのレビューコメント対応を指示された場合、または本スキルが明示的に呼び出された場合に起動してください。

---

## 📋 実行プロセス（手順）

### 1. レビューコメント一覧の取得
- GitHub MCPサーバー（`pull_request_read`の`get_review_comments`）を用いて、対象PRの全レビュースレッド・コメントを取得します。
- 1スレッドに複数コメントが含まれる場合、それぞれ別の指摘として扱います（後述の通り、原則1コメント=1コミットのため）。

### 2. コメントごとの対応方針の判断
各コメントについて、以下のいずれかに分類し対応します。

- **修正が必要と判断した場合**: 手順3（実装）に進む。
- **質問（コードの意図確認等）の場合**: 実装を読んで正確な回答を用意する。曖昧な場合は下記「ユーザーへの確認」に従いユーザー自身に確認してから回答する。
- **修正の必要が無いと判断した場合**（指摘は理解できるが、設計上の理由で現状維持が妥当と判断した場合等）: その理由を明確に言語化する。

**大きな設計判断（データ型の変更、テスト方針の変更等）を伴う指摘は、実装前にチャットで自分の見解を説明し、ユーザーの同意を得てから進めること。** 全指摘を機械的に受け入れて実装するのではなく、技術的に妥当かどうかを自分で判断し、疑問があれば説明・質問を返すこと。

### 3. 実装（修正が必要な場合）
- 既存プロジェクトルール（AGENTS.md / rules.md / test_rules.md）を必ず遵守します。
  - TDD（Red-Green-Refactor）に従って実装すること。
  - 実装・README.md・specs/以下の仕様書のいずれかに修正が生じた場合、3者間の乖離がないか確認し、乖離があれば連動して修正すること。
  - 上記の連動修正が発生した場合、CHANGELOG.mdに変更内容を記録すること。
- 修正後、[commit_rules.md](../../../commit_rules.md)記載の検証コマンド（テスト・lint・型チェック等）を実行し、全て成功することを確認します。

### 4. コミット（1コメント=1コミットの原則）
- **必ず[auto-commitスキル](../auto-commit/SKILL.md)を経由してコミットします。本スキルおよびAIエージェントが直接`git commit`を実行してはいけません。**
- **原則として1つのレビューコメントに対して1つのコミットを作成します。** 複数コメントをまとめて1コミットにしないこと。
  - 例外: 複数のコメントが実質的に同じ内容で、対応した結果の差分が完全に同一になる場合はまとめてよい。
- auto-commitスキルの手順（差分確認→検証→ユーザーへのY/N確認→コミット実行）を1コメントごとに毎回行う。

### 5. コメントへの返信
- コミットが完了したら、そのコミットのハッシュへのGitHub上のリンク（`https://github.com/<owner>/<repo>/commit/<コミットハッシュ>`）を含めて、対応した内容を`add_reply_to_pull_request_comment`で該当コメントに返信します。
- 質問への回答、または修正不要と判断した場合は、その理由・回答を同様に該当コメントへ返信します。
- 1つのコメントへの対応（実装orコミュニケーション）が完了してから、次のコメントに着手すること（並行して複数コメントの実装を進めない）。

### 6. 全コメント対応後
- 全コメントへの対応・返信が完了したら、ユーザーに完了を報告します。
- ユーザーから本ワークフローのスキル化を依頼された場合、このSKILL.md自体を新規作成・更新します。

### 7. 🚫【警告】プッシュの絶対禁止
- **本スキル、あるいはAI自身の手によって`git push`を実行することは絶対に禁止されています。** リモートリポジトリへの送信は必ずユーザー（人間）自身が行う必要があります。
